# Stats Date Range Spec

## Why
当前统计页日期选择仅支持单选（日粒度选一天、月粒度选一个月），用户无法查看跨日/跨月区间的统计数据。需要支持区间选择。

## What Changes
- 日粒度和月粒度支持选择两个日期/月份，统计区间内的数据
- Store 中 `selectedDate` 替换为 `selectedRange: { start, end } | null`
- `filterByGranularity` 支持区间过滤
- 日历 UI 支持区间选择（点击起点 → 点击终点）
- 周粒度和年粒度保持原单选行为不变
- 所有统计卡片、趋势图、任务分布随区间范围重新计算

## Impact
- Affected specs: flowtime-app（统计详情需求）
- Affected code:
  - `src/stores/useStatsStore.ts` — 状态模型变更
  - `src/utils/stats.ts` — `filterByGranularity` 扩展
  - `src/pages/Stats.tsx` — 日历选择器 UI + 数据显示

## MODIFIED Requirements

### Requirement: 统计详情（修改）
系统 SHALL 在日粒度和月粒度下支持区间选择（起止两个日期/月份），统计该区间内的所有数据。

#### 区间选择 vs 单选
| 粒度 | 区间选择 | 单选（保留） |
|------|---------|-------------|
| 日 | 选择起止两个日期，统计含头含尾的所有数据 | 点击同一日期两次 = 单选该日 |
| 月 | 选择起止两个月份，统计含头含尾的所有数据 | 点击同一月份两次 = 单选该月 |
| 周 | 不支持区间，保持单选整周 | 点击即选中该周 |
| 年 | 不支持区间，保持单选整年 | 点击即选中该年 |

#### Scenario: 日粒度区间选择
- **WHEN** 用户在日粒度下打开日历
- **AND** 点击第一个日期
- **THEN** 该日期高亮为起点（蓝色圆环或边框标记）
- **AND** 底部显示提示文案"请选择结束日期"
- **WHEN** 用户点击第二个日期
- **THEN** 起止日期之间的所有日期高亮为区间（浅紫色背景）
- **AND** 弹窗自动关闭，统计数据刷新为该区间数据
- **AND** 按钮标签更新为区间格式（如 "6/1 - 6/7"）

#### Scenario: 月粒度区间选择
- **WHEN** 用户在月粒度下打开日历
- **AND** 点击第一个月份
- **THEN** 该月份高亮为起点
- **AND** 底部显示提示文案"请选择结束月份"
- **WHEN** 用户点击第二个月份
- **THEN** 起止月份之间的所有月份高亮为区间
- **AND** 弹窗自动关闭，统计数据刷新为该区间数据
- **AND** 按钮标签更新为区间格式（如 "2026年1月 - 3月"）

#### Scenario: 起点晚于终点自动交换
- **WHEN** 用户选择的第二个日期/月份在第一个之前
- **THEN** 自动交换，早的作为起点、晚的作为终点

#### Scenario: 同日期/同月 = 单选
- **WHEN** 用户两次点击同一日期或同一月份
- **THEN** 视为单选该日期/月份（退化为原行为）
- **AND** 按钮标签显示单选格式（如 "2026年6月7日"）

#### Scenario: 重新选择
- **WHEN** 区间选择完成后，用户再次打开日历
- **AND** 点击新日期
- **THEN** 上一次的区间选中状态清除，开始新的起点选择

#### Scenario: 清除区间（返回今天）
- **WHEN** 用户已选择区间范围
- **THEN** 日历按钮旁显示"清除筛选"链接
- **AND** 点击后恢复为当前时间范围（selectedRange = null），表现同"返回今天"

#### Scenario: 周/年粒度无区间
- **WHEN** 用户在周或年粒度下打开日历
- **THEN** 日历选择器行为不变（单选整周/整年）
- **AND** 无"请选择结束日期"提示

#### Scenario: 趋势图在区间下的行为
- **WHEN** 用户在日粒度下选择了多日区间（如 6/1 - 6/7）
- **THEN** 专注趋势图 X 轴仍为 24 小时（聚合整个区间内每天的对应小时数据）
- **WHEN** 用户在月粒度下选择了多月区间（如 1月 - 3月）
- **THEN** 专注趋势图 X 轴为 3×4=12 周（每月的每周各自独立显示）

#### Scenario: 概览卡片在区间下重新计算
- **WHEN** 区间选择生效
- **THEN** 总专注时长 = 区间内所有已完成 work 记录的 actual_duration 之和
- **AND** 完成番茄数 = 区间内 completed 的 work 记录数
- **AND** 日均番茄数 = 完成番茄数 / 区间天数（日粒度）或 / (月份数×当月天数)（月粒度）
- **AND** 日均专注时长 = 总专注时长 / 区间天数（日粒度）或 / (月份数×当月天数)（月粒度）
- **AND** 完成率 = 区间内 completed 数 / 区间内 work 总数
